Introduction

The S2LET code provides high performance routines for performing wavelet analysis of signals given on the sphere. It was initially intended to work with the SSHT code that uses the MW sampling (ArXiv | DOI) to perform exact spherical harmonic transform on the sphere. In that case the wavelet transform is also theoretically exact, i.e. it captures all the information of band-limited signals in the minimal number of samples. S2LET was extended in order to support the HEALPIX format, in which case the transforms are not theoretically exact.

This documentation outlines the main features of S2LET, installation details as well as the core functionalties and interfaces. Reference, version, and license information then follows. For an overview see the README.txt file. S2LET requires the SSHT, FFTW and CFITSIO packages. To support HEALPIX, a valid installation of its Fortran implementation must be provided.

Wavelet transform on the sphere

Following the needlet approach, wavelet kernels are constructed through exact tilling of harmonic space. With an exact spherical harmonic transform as the one provided in SSHT, the wavelet transform is theoretically exact in both pixel and harmonic spaces. S2LET then achieves floating-point precision. When using HEALPIX instead of the MW sampling, the harmonic transform is not exact and hence the corresponding wavelet transform performed by S2LET is only exact in harmonic space. In other words, a set of spherical harmonic coefficients may be projected onto wavelets exactly in harmonic spaces but the HEALPIX maps don't capture all the information contained in these coefficients. However it achieves good numerical accuracy in a wide range of situations.

S2LET provides user-friendly routines for performing the wavelet transform in both real and harmonic space. Moreover, since the individual wavelet kernels have different band-limits, they may be reconstructed at different resolutions on the MW sampling. For this reason all MW routines are implemented in two versions : the usual transform where all maps are constructed at full resolution on the sphere, and a multiresolution algorithm that stores each wavelet map in the number of samples corresponding to its band-limit. This multiresolution approach fully exploit the advantages of the tilling of harmonic space as well as the exactness of both the spherical harmonic and the wavelet transforms. The multiresolution cannot be used with HEALPIX as its spherical harmonic transform is not exact.

Installation

The core functionalities of S2LET are written in C and self-documented. Matlab interfaces are provided for all high-level routines. They allow to read/write FITS maps (for both the MW and HEALPIX formats), compute the spherical harmonic and the wavelet transforms and plot the resulting signals on the sphere. To support HEALPIX, S2LET uses a hybrid C/Fortran interface to the Fortran HEALPIX library. Several examples, tests and demos are provided for both the C and Matlab codes.

Dependencies

S2LET requires the SSHT, FFTW and CFITSIO packages. Obviously suitable C and Fortran 90 compilers will be required to build the respective implementations. Both C and Matlab mex compilers will be required to build the Matlab interface to the C implementation. - you *can* build LIB without CFITSIO or HEALPIX -> all interfaces work, but you handle the IO part - linking to CFITSIO adds MW input-output - linking to HEALPIX adds HPX-SHA and HPX input-output

Compiling

Makefiles are provided to build the code. However the locations of SSHT, FFTW, CFITSIO and optionnaly HEALPIX must be specified. In S2LET the default Makefile will look at these symbols in Bash/Shell profiles. Alternatively you may write them explicitely in the first section of the makefile. The compilers and their options may also be changed there.

Once you have set the makefile up for your system, S2LET may be compiled by running: 

 >> make all

This will build the C library and Matlab interface, the documentation and tests. The HEALPIX support will be build if HEALPIX is found. The test programs will run automatically so that you can see of the code was compiled properly. For the MW sampling the wavelet transform should achieve floating-point precision.

You may alternatively build the individual components of SSHT. To build the library only run: 

 >> make lib

To build the test framework for the exact wavelet transform (MW sampling, default) run: 

 >> make test

To build the test framework for the HEALPIX wavelet transform (optional) run: 

 >> make hpxtest

To build the Matlab interfaces (MW by default, HEALPIX if found): 

 >> make matlab

A demo of the wavelet transform on a CMB simulation (HEALPIX map) is also provided, and can be build with: 

 >> make demo

Tests and programs

s2let_test s2let_hpxtest s2let_hpxiodemo s2let_axisym_hpx_analysis_real s2let_axisym_mw_analysis_real

Three programs may be run to test the framework. They are extensively documented and may serve as starting point to learn to use the functionalities of S2LET. The command 

 >> ./bin/c/s2let_test
will run various performance and accuracy tests for the exact Wavelet transform built on the MW sampling. All algorithms and high-level features are tested, including the pixel space and harmonic space transforms with and without the multiresolution algorithm.

If HEALPIX is provided, the command 

 >> ./bin/c/s2let_hpxtest
will test the interfaces to the Fortran implementation of HEALPIX as well as the corresponding wavelet transform.

Finally, a full demo is provided to read a HEALPIX map of a random CMB simulation, perform its wavelet decomposition and write the wavelet maps to FITS files in the data directory. 

 >> ./bin/c/s2let_demo

To check the version and build numbers of your version run: 

 >> ./bin/c/s2let_about

Documentation

The S2LET code is self documenting. Although the package ships with source documentation, you may generate html source documentation by running: 

 >> make doc

Documentation is generated using doxygen, thus you must have doxygen installed on your system to generate the source documentation.

Cleaning up

To tidy up all temporary files and objects run: 

 >> make tidy

To remove all compiled files and programs run: 

 >> make clean

To remove all source documentation run: 

 >> make cleandoc

Source code documentation

S2LET ships with source documentation that is generated by doxygen. Documentation is available here. The Matlab routines that interface with the C implementation are self documenting (documentation can be access through the help command in Matlab), as discussed below.

Matlab interface

Once the Matlab interface is built, you must have s2let/src/matlab in your path in order to run the Matlab functions. A number of Matlab functions and demos illustrating their use are provided.

Functions

A full list of Matlab functions, with brief descriptions are given here: 


  s2let_axisym_analysis
  Compute exact wavelet transform of a signal sampled on the sphere (MW sampling)

  s2let_axisym_synthesis
  Exactly reconstuct a signal on the sphere from its wavelet contributions (MW sampling)

  s2let_axisym_tilling
  Compute the tilling of harmonic space

  s2let_check_axisym_tilling
  Check exactness (i.e. identity resolution) of the harmonic tilling

  s2let_hpx_alm2map
  Interface to the Fortran function alm2map in HEALPIX (ring ordered, inverse spherical harmonic transform)

   s2let_hpx_map2alm
  Interface to the Fortran function map2alm in HEALPIX (ring ordered, forward spherical harmonic transform)

  s2let_hpx_axisym_analysis
  Compute wavelet transform of a HEALPIX map (ring ordered)

  s2let_hpx_axisym_synthesis
  Reconstuct a HEALPIX map (ring ordered) on the sphere from its wavelet maps

   s2let_plot_hpx_mollweide
  Plot a HEALPIX map (ring ordered) using Mollweide projection

To access the documentation for each function in Matlab, simply run: 

 >> help <function>

where <function> is the function name.

Demos

A number of demos are provided to illustrate the use of the Matlab interface. 

  s2let_fulltest
  Run and test all functionalties of the wavelet transform (MW sampling)

  s2let_hpxtest
  Run and test all functionalties of the wavelet transform (HEALPIX sampling)

  s2let_demo1
  Compute and plot wavelets of Earth tomography data (full and multi-resolution, MW sampling)

  s2let_demo2
  Plot wavelet kernels corresponding to some set of parameters

  s2let_demo3
  Compute and plot wavelets of a random CMB simultion (HEALPIX sampling)

To access the documentation for each demo in Matlab, simply run: 

 >> help <demo>

where <demo> is the demo name.
