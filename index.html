Introduction

The S2LET code provides high performance routines for fast wavelet analysis of signals given on the sphere. It is primarily intended to work with the SSHT code that uses the MW sampling (ArXiv | DOI) to perform exact spherical harmonic transform on the sphere. In that case the wavelet transform is theoretically exact, i.e. it captures all the information of band-limited signals in the minimal number of samples. S2LET was also extended to support the HEALPIX format, in which case the transforms are not theoretically exact but achieve good numerical accuracy.

This page outlines the main features of S2LET, installation details as well as the core functionalties and interfaces. References, version, and license information then follows. S2LET requires the SSHT and FFTW libraries. The IO FITS features require CFITSIO. To support HEALPIX, a valid installation of its Fortran implementation must be provided.


Wavelet transform on the sphere

In S2LET, the wavelet kernels are constructed through exact tilling of harmonic space, as shown on the image below for a particular set of wavelet parameters (B=2, J_min=2, detailed in the relevant publications). Harmonic space is tiled, and the corresponding wavelet kernels are localised (i.e. have compact support) in both real and frequency spaces.

      

With an exact spherical harmonic transform as the one provided in SSHT, the wavelet transform is theoretically exact in both pixel and harmonic spaces. S2LET then achieves floating-point precision, and one can decompose a MW map in a set of wavelet maps that exactly capture all the information. Using the previous tiling of harmonic space, the individual wavelet maps have different band-limits and hence can be reconstructed at different resolutions, i.e. on the minimal number of samples. The left and right panels below are the wavelet decomposition of Earth tomography data at full (left) and multi-resolution (right). Due to the exactness of all transforms, these two approaches are equivalent and capture all the information contained in the initial band-limited map.

         

When using HEALPIX instead of the MW sampling, the harmonic transform is not exact and hence the corresponding wavelet transform performed by S2LET is only exact in harmonic space. In other words, a set of spherical harmonic coefficients may be projected onto wavelets exactly in harmonic spaces but the HEALPIX maps don't capture all the information contained in these coefficients. However it achieves good numerical accuracy in a wide range of situations.


Installation and documentation

The core functionalities of S2LET are written in C and self-documenting. The C library only requires SSHT and FFTW. The extra IO and Healpix features require the CFITSIO and HEALPIX libraries. Interfaces are provided for all high-level routines in Matlab, IDL and Java. In Matlab and IDL, these interfaces allow to read/write FITS maps (for both the MW and HEALPIX formats), compute the spherical harmonic and the wavelet transforms and plot the resulting signals on the sphere. To support HEALPIX, S2LET uses a hybrid C/Fortran interface to the Fortran HEALPIX library. Several examples, tests and demos are provided fo the C library and all interfaces in Matlab, IDL and Java.

Dependencies

S2LET requires the SSHT and FFTW packages. Obviously suitable C and Fortran 90 compilers will be required to build the respective implementations. Both C and Matlab mex compilers will be required to build the Matlab interface.
Linking with CFITSIO adds the FITS input-output functionalities for the MW sampling. Linking with HEALPIX adds support for the Healpix spherical harmonic transform as well as the FITS input-output routines.
